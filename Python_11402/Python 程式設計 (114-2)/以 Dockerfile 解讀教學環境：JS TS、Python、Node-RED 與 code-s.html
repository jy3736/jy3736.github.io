以 Dockerfile 解讀教學環境：JS/TS、Python、Node-RED 與 code-server

課程簡介

本講義以 resources/Dockerfile 為唯一來源，帶你逐層理解容器內已建好的學習環境。你會知道「有哪些工具」、「如何驗證可用」、「啟動後服務如何連線」，並用 Python 驗證腳本做自動檢查，降低環境問題造成的學習阻力。

本講義涵蓋主題

1. 基底映像與系統套件層

2. Node.js 與 JavaScript 開發工具鏈（以 Python 驗證）

3. Python 虛擬環境與資料分析工具

4. code-server、JupyterLab、Node-RED 服務整合

5. 使用者、目錄、連接埠與啟動流程

6. 綜合實作：3 支 Python 環境健康檢查腳本

1. 基底映像與系統套件層

A. 概念說明

Dockerfile 使用 ubuntu:latest 作為基底，並一次安裝編譯、除錯、版本控制與網路工具。這一層決定了後續語言環境能否順利建置與執行。

B. 範例程式碼

原始範例：檢查作業系統與核心工具是否存在。

import subprocess

os_name = subprocess.check_output("grep PRETTY_NAME /etc/os-release", shell=True, text=True).strip()
git_path = subprocess.check_output("command -v git", shell=True, text=True).strip()
print(os_name)
print(f"git path: {git_path}")

補強範例 1：批次檢查編譯工具。

import shutil

for cmd in ["gcc", "g++", "cmake", "ninja", "gdb", "valgrind"]:
    print(f"{cmd}: {'OK' if shutil.which(cmd) else 'MISSING'}")

補強範例 2：讀取工具版本資訊。

import subprocess


def version(cmd):
    try:
        out = subprocess.check_output(cmd, shell=True, text=True)
        return out.splitlines()[0]
    except Exception:
        return "not available"


print("clang ->", version("clang --version"))
print("lldb  ->", version("lldb --version"))

C. 注意事項

- ubuntu:latest 會隨時間變動，建議教學正式環境改用固定 tag。

- 套件安裝集中在單一 RUN，可減少 layer，但除錯時可讀性較低。

2. Node.js 與 JavaScript 開發工具鏈（以 Python 驗證）

A. 概念說明

Dockerfile 透過 NodeSource 安裝 Node.js 20，並全域安裝 typescript、ts-node、eslint、prettier、nodemon、yarn、pnpm、node-red 等工具。雖然目標是 JS/TS 開發，本章以 Python 進行一致化驗證。

B. 範例程式碼

原始範例：檢查 Node 與 npm 版本。

import subprocess

node_v = subprocess.check_output("node -v", shell=True, text=True).strip()
npm_v = subprocess.check_output("npm -v", shell=True, text=True).strip()
print({"node": node_v, "npm": npm_v})

補強範例 1：檢查常用全域 CLI。

import shutil

for tool in ["tsc", "ts-node", "eslint", "prettier", "nodemon", "pnpm", "yarn"]:
    path = shutil.which(tool)
    print(f"{tool}: {path if path else 'missing'}")

補強範例 2：驗證 Node.js 可執行。

import subprocess

runtime = subprocess.check_output('node -e "console.log(JSON.stringify({v:process.version,p:process.platform}))"', shell=True, text=True).strip()
print("node runtime check:", runtime)

C. 注意事項

- 全域安裝方便教學，但專案實務仍建議以本地依賴鎖版本。

- node-red-dashboard 等套件版本會更新，課程應定期回歸測試流程。

3. Python 虛擬環境與資料分析工具

A. 概念說明

Dockerfile 建立 /opt/venv 並將其加入 PATH，安裝 JupyterLab、pandas、numpy、matplotlib、pytest、black 等工具。這讓同一容器可同時支援 JS 與 Python 實作。

B. 範例程式碼

原始範例：檢查 Python 版本。

import subprocess

print(subprocess.check_output("python -V", shell=True, text=True).strip())

補強範例 1：確認 Python 來自 venv。

import subprocess

python_path = subprocess.check_output("which python", shell=True, text=True).strip()
print("python path:", python_path)
print("is venv:", python_path.startswith("/opt/venv/"))

補強範例 2：檢查主要 Python 套件可匯入。

import importlib

for pkg in ["numpy", "pandas", "matplotlib", "jupyterlab"]:
    try:
        importlib.import_module(pkg)
        print(f"{pkg}: OK")
    except Exception:
        print(f"{pkg}: MISSING")

C. 注意事項

- Dockerfile 內 pip install pandas 重複出現，不影響功能但會增加建置時間。

- 若課程不需部分 Python 工具，可拆分成精簡版映像降低映像體積。

4. code-server、JupyterLab、Node-RED 服務整合

A. 概念說明

環境在啟動時同時開三個服務：

- code-server：8080

- JupyterLab：8888

- Node-RED：1880

ENTRYPOINT 以 shell 串接三個命令，使單一容器具備 IDE、Notebook、Flow-based 開發能力。

B. 範例程式碼

原始範例：列出預期服務連線網址。

services = {
    "code_server": "http://localhost:8080",
    "jupyter_lab": "http://localhost:8888",
    "node_red": "http://localhost:1880",
}
for k, v in services.items():
    print(f"{k}: {v}")

補強範例 1：檢查連接埠是否可連線。

import socket


def check_port(port, host="127.0.0.1"):
    with socket.socket(socket.AF_INET, socket.SOCK_STREAM) as s:
        s.settimeout(1.5)
        return s.connect_ex((host, port)) == 0


for p in [8080, 8888, 1880]:
    print({"port": p, "ok": check_port(p)})

補強範例 2：用 HTTP 探測服務狀態碼。

import urllib.request

for url in ["http://127.0.0.1:8080", "http://127.0.0.1:8888", "http://127.0.0.1:1880"]:
    try:
        with urllib.request.urlopen(url, timeout=2) as resp:
            print({"url": url, "status": resp.status})
    except Exception:
        print({"url": url, "status": "down"})

C. 注意事項

- 目前 code-server 與 Jupyter token/password 設為空，僅適合內網或教學封閉環境。

- 單容器多服務方便教學，但正式部署建議拆成多容器並加反向代理與認證。

5. 使用者、目錄、連接埠與啟動流程

A. 概念說明

Dockerfile 建立 stust 使用者、預設工作目錄 /home/stust，並初始化 workspace 與設定檔。這有助於學生登入後立即開始開發，不需要手動初始化 IDE 與資料夾。

B. 範例程式碼

原始範例：檢查目前登入使用者與工作目錄。

from pathlib import Path
import getpass
import os

print("user:", getpass.getuser())
print("home:", str(Path.home()))
print("cwd:", os.getcwd())

補強範例 1：確認關鍵目錄存在。

from pathlib import Path

for p in [
    "/home/stust/workspace",
    "/home/stust/.node-red",
    "/home/stust/.local/share/code-server",
]:
    print(f"{p}: {'exists' if Path(p).exists() else 'missing'}")

補強範例 2：檢查 README 是否可讀取。

from pathlib import Path

readme = Path("/home/stust/workspace/README.md")
if readme.exists():
    preview = "\n".join(readme.read_text(encoding="utf-8").splitlines()[:3])
    print(preview)
else:
    print("README not found")

C. 注意事項

- stust:stust 密碼僅供教學，實務上必須改密碼與停用弱認證。

- ENTRYPOINT 以 sh -c 背景啟動多服務，若其中一個服務異常可能不易即時觀察。

6. 綜合實作：容器學習環境健康檢查器

A. 實作目標

1. 提供 3 支獨立驗證腳本：JS/TS、Python、Node-RED。

2. 每支腳本輸出 PASS/FAIL，並可作為課前自檢工具。

B. 完整程式碼範例（3 支驗證腳本）

1. validate_js_ts.py

import json
import shutil
import subprocess
import sys


def safe_exec(cmd):
    try:
        return subprocess.check_output(cmd, shell=True, text=True).strip()
    except Exception:
        return "missing"


checks = {
    "node": safe_exec("node -v"),
    "npm": safe_exec("npm -v"),
    "tsc": safe_exec("tsc -v"),
    "ts_node": safe_exec("ts-node -v"),
    "eslint": safe_exec("eslint -v"),
    "prettier": safe_exec("prettier -v"),
    "node_path": shutil.which("node") or "missing",
}

failed = any(v == "missing" for v in checks.values())
print(json.dumps({"target": "js-ts", "checks": checks}, indent=2, ensure_ascii=False))
if failed:
    sys.exit(1)
print("PASS: JS/TS env is ready")

2. validate_python_env.py

import json
import subprocess
import sys


def safe_exec(cmd):
    try:
        return subprocess.check_output(cmd, shell=True, text=True).strip()
    except Exception:
        return "missing"


checks = {
    "python": safe_exec("python -V"),
    "pip": safe_exec("pip -V"),
    "jupyter": safe_exec("jupyter --version"),
    "numpy": safe_exec('python -c "import numpy; print(numpy.__version__)"'),
    "pandas": safe_exec('python -c "import pandas; print(pandas.__version__)"'),
    "matplotlib": safe_exec('python -c "import matplotlib; print(matplotlib.__version__)"'),
}

failed = any(v == "missing" for v in checks.values())
print(json.dumps({"target": "python", "checks": checks}, indent=2, ensure_ascii=False))
if failed:
    sys.exit(1)
print("PASS: Python env is ready")

3. validate_node_red.py

import json
import socket
import subprocess
import sys


def safe_exec(cmd):
    try:
        return subprocess.check_output(cmd, shell=True, text=True).strip()
    except Exception:
        return "missing"


def check_port(port, host="127.0.0.1"):
    with socket.socket(socket.AF_INET, socket.SOCK_STREAM) as s:
        s.settimeout(1.5)
        return s.connect_ex((host, port)) == 0


checks = {
    "node_red_cmd": safe_exec("node-red --version"),
    "node_red_port_1880": check_port(1880),
}

print(json.dumps({"target": "node-red", "checks": checks}, indent=2, ensure_ascii=False))
if checks["node_red_cmd"] == "missing" or not checks["node_red_port_1880"]:
    sys.exit(1)
print("PASS: Node-RED env is ready")
