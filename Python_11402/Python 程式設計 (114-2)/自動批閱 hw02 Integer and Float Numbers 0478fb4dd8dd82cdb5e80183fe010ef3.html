2. 核心概念群一：整數、浮點數與型別轉換

A. 定義式解釋

整數 int 用來表示沒有小數部分的數值，例如人數、分鐘數、糖果數。浮點數 float 用來表示可能有小數的數值，例如單價、半徑、平均成本。Python 的 input() 讀進來一律是文字，所以若題目要做數值運算，通常必須先轉型。

B. 題目脈絡式解釋

prob01 同時出現整數的分鐘數與浮點數的時薪，這是在提醒學生：不同資料可能要用不同型別處理。prob05 則更典型，因為人數與每組上限是整數，但成本是浮點數，三者不能全部用同一種想法對待。

C. 解題策略式解釋

看到輸入格式時，先畫一條小表格：每個欄位叫什麼、應該是整數還是浮點數、之後要參與哪種運算。只要型別判斷先做對，後面的公式通常不難。

D. 常見錯誤與修正

- 把 input() 讀到的字串直接拿去除法，沒有先轉成 int() 或 float()。

- 明明題目有金額或半徑，卻全部用 int()，導致小數資訊被破壞。

- 不知道 / 的結果通常會變成浮點數，所以輸出格式跟著出錯。

minutes_text, rate_text = input().split()
minutes = int(minutes_text)
rate = float(rate_text)

hours = minutes / 60
pay = hours * rate

這段在做的是先把輸入拆成兩段文字，再依照資料性質轉成整數與浮點數。為什麼要這樣寫？因為分鐘是計數資料，時薪可能有小數。初學者最容易不懂的地方是：不是所有數字都該用 int()，要看題目要保留什麼資訊。

3. 核心概念群二：除法、整除、餘數與分配思維

A. 定義式解釋

/ 是一般除法，結果通常是浮點數。// 是整除，用在只想知道「完整能分成幾份」的情況。% 是餘數，用在想知道「分完還剩多少」的情況。

B. 題目脈絡式解釋

prob01 用 / 把分鐘換成小時，因為結果可能是 1.5 小時。prob02 用 // 和 % 把糖果分給小朋友，因為題目要的是每人拿到幾顆與剩幾顆。prob05 則進一步告訴學生：有時候只靠 // 還不夠，還要搭配 ceil() 才能算出需要幾組。

C. 解題策略式解釋

遇到分配題時，先問自己題目到底要哪一種答案。若是「平均值」或「實際比例」，先想 /；若是「完整份數」，先想 //；若是「剩下多少」，先想 %。如果題目問「至少要幾組」或「至少要幾個容器」，通常還要想到向上取整。

D. 常見錯誤與修正

- 把 candies / kids 當成每人可拿的糖果數，結果得到小數。

- 把 people // max_per_group 當成組數，忽略有剩人時還要多一組。

- 看見 % 只會背公式，卻不知道它是在表示「剩下的部分」。

candies, kids = map(int, input().split())
each = candies // kids
left = candies % kids

print(each)
print(left)

這段是在做平均分裝。each 表示每人拿到的完整顆數，left 表示分完後剩下的糖果。新手容易看不懂的是：// 和 % 常常成對出現，因為一個管「完整份數」，另一個管「剩餘量」。

4. 核心概念群三：冪次、公式套用與 math 模組

A. 定義式解釋

** 是冪次運算，例如 2 ** 5 代表 2 的 5 次方。math.pi 是圓周率常數，math.ceil(x) 則會把數值向上取整到最接近的整數。

B. 題目脈絡式解釋

prob03 幾乎只在檢查學生是否理解 2^n 對應 Python 的 2 ** n。prob04 則把數值運算推進一步，不只要寫公式，還要記得先 import math，並理解 area、circ、tiles 各自代表不同型別與不同輸出要求。

C. 解題策略式解釋

看到數學題時，先別急著寫程式，先把公式抄成變數關係。確認每個變數是用來表示面積、周長、次方結果，還是向上取整後的整數。這樣能把題目從「看起來很多字」轉成「只是幾個變數與公式」。

D. 常見錯誤與修正

- 把 2^n 寫成 2 ^ n，但在 Python 裡 ^ 不是冪次。

- 忘了先 import math 就直接使用 math.pi 或 math.ceil()。

- 把 tiles 也格式化成小數，違反題目要整數的要求。

import math

radius = float(input())
area = math.pi * radius ** 2
circ = 2 * math.pi * radius
tiles = math.ceil(area)

這段在做的是先算圓面積與周長，再估算覆蓋面積至少需要幾塊瓷磚。為什麼 tiles 要用 ceil()？因為只要多出一點點面積，還是需要再多一塊完整瓷磚。初學者常卡住的地方是：前兩個結果是小數，最後一個卻是整數，三者的輸出方式不一樣。

5. 核心概念群四：混合題中的邊界條件與拆解順序

A. 定義式解釋

混合題是指題目同時包含多個步驟與多種運算，不能只靠一條公式直接完成。這時必須把問題拆成中間變數，一步一步算。

B. 題目脈絡式解釋

prob05 就是代表題。它同時要學生算組數、最後一組人數、平均成本，還要區分「剛好整除」和「有剩人數」兩種情況。這題雖然不長，但很適合訓練拆步驟的能力。

C. 解題策略式解釋

先從最穩定的量開始算，例如組數 groups。再決定最後一組人數 last_group。最後再用已經算好的 groups 去求平均成本。不要三件事一起想，會把條件弄亂。

D. 常見錯誤與修正

- people % max_per_group == 0 時，錯把最後一組人數寫成 0。

- 先算 avg_cost，但前面的 groups 還沒處理好。

- 雖然公式對了，卻忘了平均成本要保留 2 位小數。

import math

people, max_per_group, total_cost_text = input().split()
people = int(people)
max_per_group = int(max_per_group)
total_cost = float(total_cost_text)

groups = math.ceil(people / max_per_group)

if people % max_per_group == 0:
    last_group = max_per_group
else:
    last_group = people % max_per_group

這段是在示範如何把混合題拆成小步驟。先算組數，再判斷最後一組的人數。新手最常不懂的是：餘數為 0 不代表最後一組沒有人，而是代表每組都剛好滿，所以最後一組仍然是 max_per_group 人。

6. sample code 與 code snippets 導讀

A. 讀入整數與浮點數

minutes, rate = input().split()
minutes = int(minutes)
rate = float(rate)

這段在做的是先讀入兩個值，再依照資料性質轉型。為什麼這樣寫？因為 input() 原本讀到的是文字，不能直接假設它已經是數字。新手常見錯誤是只轉第一個值，第二個值忘了轉型。

B. 保留小數點後 2 位

hours = 1.5
print(f"{hours:.2f}")

這段在做的是固定輸出格式。白話來說，就是不管原本是 1.5 還是 1.5000001，都印成兩位小數。初學者容易不懂的是 :.2f 不是數學運算，而是輸出格式控制。

C. 整除與餘數一起看

people = 17
group_size = 5

full_groups = people // group_size
left_people = people % group_size

這段在做的是同時觀察「完整分成幾組」和「剩下幾人」。為什麼要一起看？因為很多分組題同時需要這兩種資訊。新手容易只算其中一個，導致後面條件不完整。

D. 冪次運算

n = 10
value = 2 ** n
print(value)

這段在做的是計算 2 的 10 次方。為什麼這樣寫？因為 Python 的冪次是 **。初學者容易寫成 2 ^ n，但那在 Python 代表另一種位元運算，不是這題要的意思。

E. math.ceil() 的用途

import math

groups = math.ceil(17 / 5)
print(groups)

這段在做的是把 3.4 組向上取整成 4 組。白話來說，只要還有剩人，就要再開一組。新手容易誤會成四捨五入，但 ceil() 不是四捨五入，而是一律往上。

9. 解題引導與學習建議

讀題時建議先圈出三件事：輸入有哪些值、每個值是整數還是浮點數、輸出有沒有固定格式。對這組題目來說，很多錯誤不是公式不會，而是型別和輸出沒控制好。

實作時建議採用這個順序：

1. 先把輸入轉成正確型別。

2. 用中間變數寫出公式，不要把所有運算塞進 print()。

3. 最後才處理輸出順序與小數位數。

除錯時可以做這些事：

- 先印出轉型後的變數，確認值真的對。

- 用題目範例以外的邊界值測試，例如 0、剛好整除、剛好多出 1 人。

- 若題目要求 import math，先確認真的有匯入，再執行公式。

極短示意片段

# 僅用於說明拆解順序，不是完整答案
value_1, value_2 = input().split()
value_1 = int(value_1)
value_2 = float(value_2)

result = value_1 / value_2
print(f"{result:.2f}")

這段只是示範最基本的流程：讀入、轉型、運算、格式化輸出。先看懂流程，再回到各題目的實際公式。
