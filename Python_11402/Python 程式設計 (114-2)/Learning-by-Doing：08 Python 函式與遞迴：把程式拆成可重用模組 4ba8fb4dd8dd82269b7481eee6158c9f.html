5. 綜合實作：階層費率與遞迴計算工具

A. 實作目標

1. 建立可重複執行的資料處理流程。

2. 產生可讀的摘要輸出，便於驗證結果。

B. 逐步構建（Learning-by-Doing）

1. Step 1：建立資料結構

records = [
    {"name": "Amy", "value": 72},
    {"name": "Ben", "value": 88},
    {"name": "Cody", "value": 91},
]
print(records)

2. Step 2：加入核心流程

total = 0
for r in records:
    total += r["value"]
avg = total / len(records)
print(f"平均值: {avg:.2f}")

3. Step 3：完成輸出/計算

top = max(records, key=lambda r: r["value"])
print(f"最佳表現: {top['name']} ({top['value']})")

C. 完整程式碼範例

def analyze(records):
    total = sum(r["value"] for r in records)
    avg = total / len(records) if records else 0
    top = max(records, key=lambda r: r["value"]) if records else None
    return {"total": total, "avg": avg, "top": top}


def main():
    records = [
        {"name": "Amy", "value": 72},
        {"name": "Ben", "value": 88},
        {"name": "Cody", "value": 91},
    ]
    result = analyze(records)
    print(f"總分: {result['total']}")
    print(f"平均: {result['avg']:.2f}")
    if result["top"]:
        print(f"最高: {result['top']['name']} ({result['top']['value']})")


if __name__ == "__main__":
    main()

D. 程式碼亮點解析

1. 函式化與可測試性

result = analyze(records)

把計算邏輯集中在 analyze()，可以單獨測試，避免把所有邏輯都塞在主流程。

擴展練習

- 把綜合實作改成「可重複輸入多筆資料」的互動版本。

- 加入例外處理（try/except）避免輸入格式錯誤。

- 將核心邏輯拆成 2-3 個函式，並為每個函式加上測試資料。
