自動批閱 hw03: Conditions, if / elif / else

題組簡介

這一組題目聚焦在 Python 的條件判斷。學生會從最基本的單一 if 分支，走到 and、or 的多條件組合，再進一步處理分段計價、分類判斷與多重資格審核。這是非常典型的初學者過渡單元，因為它第一次要求你不要只「照順序做事」，而是要讓程式根據輸入資料做決策。

如果學生程式基礎還很有限，這組作業最重要的學習目標不是寫得多短，而是先看懂「題目在問哪些條件」、「條件的先後順序會不會影響答案」、「哪些情況要先排除」。整組題目也明確限制不可使用迴圈，教學意圖是把注意力集中在條件式本身，而不是用重複結構掩蓋思考流程。

本篇涵蓋重點

1. 題型分類：單一判斷、邏輯組合、分段規則、分類題、多重審核題。

2. 核心概念：比較運算、布林值、if / elif / else、and / or、條件優先順序。

3. 常見卡點：邊界值、條件順序、把 = 和 == 混淆、漏掉無效情況。

4. 解題策略：先整理規則，再把條件拆成變數，最後決定輸出。

這組題目是很標準的遞進式安排。前三題 lab 先把單一條件、and、or 拆開練，讓學生熟悉「條件成立才執行」的程式心智模型。接著 prob 開始要求你同時處理數值規則與分支順序，不再只是印出一個文字。最後 extra01 則把前面所有觀念合併，加入加權分數、審核門檻與輸出格式。

建議閱讀與練習順序就是 lab01 -> lab02 -> lab03 -> prob01 -> prob02 -> prob03 -> extra01。因為後面的題目並不是新語法很多，而是規則更多、判斷更密，若前面沒有把真假條件與邊界值想清楚，後面很容易整題錯掉。

2. 核心概念群一：條件判斷的基本語法

A. 定義式解釋

if 的意思是「如果條件成立，就做這段程式」。elif 是「前面不成立時，再檢查另一個條件」。else 則是前面都不成立時的最後處理。

B. 題目脈絡式解釋

在 lab01 中，氣溫只會落在冷、暖、熱其中一種情況，所以很適合用三段式分支。這是在教學生：當題目明確把情況分成幾類時，最常見的寫法就是 if / elif / else。

C. 解題策略式解釋

看到分類題時，先不要急著寫程式。先問自己三件事：總共有幾種結果、每種結果各自對應什麼條件、是否有任何邊界值需要特別處理。這樣條件會更穩。

D. 常見錯誤與修正

- 把 20 <= temp < 30 寫成互相重疊或漏掉的條件。

- 忘記 elif 是依序檢查，順序一錯就可能把資料分到前面的分類。

- 輸入是字串卻直接比較數字，少了 int()。

temp = int(input())

if temp < 20:
    print("cold")
elif temp < 30:
    print("warm")
else:
    print("hot")

這段在做的是把輸入氣溫分成三種提醒。第二個條件直接寫 temp < 30 就夠了，因為程式能走到 elif 時，已經代表 temp < 20 不成立。初學者常看不懂的地方是「前面沒進去」其實就是一種已知資訊。

3. 核心概念群二：布林邏輯與多條件組合

A. 定義式解釋

and 表示左右條件都要成立；or 表示至少一個成立。它們的結果都是布林值，也就是 True 或 False。

B. 題目脈絡式解釋

lab02 的通過門檻同時看考試與出席，所以是 and。lab03 的休息判定只要是週末或假日其中之一即可，所以是 or。這兩題剛好在教學生區分「同時滿足」和「任一滿足」。

C. 解題策略式解釋

遇到多個條件時，先用中文口語重述一次。題目如果說「而且」，通常對應 and；題目如果說「或者」、「只要其中一個」，通常對應 or。先把語意講清楚，再翻成程式，比直接硬寫更不容易錯。

D. 常見錯誤與修正

- 把 and、or 用反，導致通過條件變太寬或太嚴。

- 把 weekend、holiday 當成字串 "1" 來比，而不是整數 1。

- 沒有先確認資料型別，條件看起來對但結果不對。

score = int(input())
attendance = int(input())

if score >= 60 and attendance >= 80:
    print("pass")
else:
    print("fail")

這段是在檢查「分數夠不夠」以及「出席夠不夠」。為什麼這樣寫？因為兩個門檻都屬於必要條件。新手最容易錯在以為只要其中一項達標就算通過，那就會把 and 誤寫成 or。

4. 核心概念群三：分段規則與分類順序

A. 定義式解釋

分段規則是把數值區間拆成幾段，每一段使用不同處理方式。這類題目通常不只是判斷真假，而是根據落點決定費率、折扣或類別。

B. 題目脈絡式解釋

prob01 的訂單折扣和 prob02 的停車費，都是典型的分段題。重點不是算術難，而是要先理解「哪一段先算、哪一段再加上去、哪些條件是後續修正」。

C. 解題策略式解釋

先把題目拆成兩層。第一層決定基本值，例如基本折扣或原始停車費；第二層再加入會員折抵、額外折扣或最低值保護。這樣結構會比把所有規則塞進同一個 if 更清楚。

D. 常見錯誤與修正

- 沒把「基本折扣」和「額外折扣」分開，導致邏輯糾纏。

- 停車費把所有小時都用同一費率算，忽略分段累加。

- 沒處理最後價格不能小於 0。

price = int(input())
member = int(input())
coupon = int(input())

if price >= 300:
    base_discount = 50
elif price >= 200:
    base_discount = 30
else:
    base_discount = 0

if member == 1 and coupon == 1:
    extra_discount = 20
elif member == 1 or coupon == 1:
    extra_discount = 10
else:
    extra_discount = 0

這段示範的是先算基本折扣，再算額外折扣。這樣寫的原因是題目本來就把規則分成兩組。初學者常卡住的地方是：明明規則看懂了，但一開始就想把全部寫在一個大條件裡，反而更容易漏情況。

5. 核心概念群四：先排除，再分類

A. 定義式解釋

有些題目不能直接分類，因為資料可能根本不合法。這時要先做驗證，再做正式判斷。這種「防呆式流程」在實務程式中非常常見。

B. 題目脈絡式解釋

prob03 要先判斷是不是有效三角形，extra01 要先判斷是否直接 rejected。這兩題都在訓練一個很重要的觀念：不是所有輸入都值得進入下一步分類。

C. 解題策略式解釋

寫條件前先找「一票否決」規則。凡是只要成立就不能繼續分類的情況，都應該先放在前面，因為它的優先權最高。

D. 常見錯誤與修正

- 先判斷等邊或等腰，卻忘了它可能根本不是三角形。

- rejected 放太後面，讓不符合資格的人被錯分成 qualified。

- 浮點分數算對了，分類順序卻錯了。

a = int(input())
b = int(input())
c = int(input())

if a <= 0 or b <= 0 or c <= 0:
    print("invalid")
elif a + b <= c or a + c <= b or b + c <= a:
    print("invalid")
elif a == b and b == c:
    print("equilateral")
elif a == b or a == c or b == c:
    print("isosceles")
else:
    print("scalene")

這段在做的是先確認三邊是否構成有效三角形，再做類型分類。為什麼先判斷 invalid？因為資料一旦無效，後面的分類就沒有意義。新手常錯在看到「三邊相等」就直接輸出 equilateral，但完全忘了邊長也可能是 0。

6. sample code 與 code snippets 導讀

A. 讀入多個整數

exam, attendance, late, project = map(int, input().split())

這段是在一次讀入四個整數。input().split() 會把一整行文字用空白切開，map(int, ...) 則把每一段文字轉成整數。初學者容易不懂的是：input() 讀進來原本都是字串，不先轉型就不能安全做數值比較。

B. 先存條件結果，再做判斷

is_member = member == 1
has_coupon = coupon == 1

if is_member and has_coupon:
    extra_discount = 20

這段是在把條件先命名。好處是可讀性更高，也更容易除錯。新手常把條件直接寫得很長，最後連自己都看不懂。

C. 浮點數格式化輸出

score = exam * 0.6 + project * 0.4
print(f"{score:.1f}")

這段是在把分數印到小數第 1 位。為什麼這樣寫？因為題目要求輸出格式固定。初學者常會直接 print(score)，結果小數位數不一致。

D. 條件骨架模板

if invalid_case:
    print("invalid")
elif special_case:
    print("special")
else:
    print("normal")

這是最通用的分類骨架。它不是完整答案，但很適合在讀題後先把思路搭出來。對初學者來說，先寫骨架比一開始追求完整更穩。

8. 解題前必備知識與能力檢核

A. 已會

- 知道 input() 讀到的是字串，需要用 int() 轉整數。

- 看得懂 >, >=, <, <=, ==, != 的意思。

- 知道 if 區塊需要正確縮排。

B. 還需補強

- 邊界值判斷，例如 20 到底算 cold 還是 warm。

- and 與 or 的語意差異。

- 多條件時的先後順序與優先規則。

- 浮點數輸出格式，例如 :.1f。

C. 容易誤解

- = 不是比較，而是指定。

- elif 不是另一個獨立的 if，它跟前面的分支有順序關係。

- 「先排除無效」不是多餘步驟，而是避免後續分類失真。

9. 解題引導與學習建議

先用中文把規則整理成清單，再決定哪些規則是先判斷、哪些規則是後判斷。對這一組題目來說，最穩的寫法通常不是最短的寫法，而是條件分段清楚、每個變數都有單一用途的寫法。

讀題時建議先問：

1. 題目要我輸出哪一種結果，是文字、整數，還是小數格式？

2. 有沒有邊界值或無效值要先處理？

3. 這些條件之間是 and、or，還是分段 elif？

除錯時建議：

- 先印出輸入值，確認轉型成功。

- 再印出每個條件的真假，確認自己理解正確。

- 最後用題目範例和自訂邊界值測試，例如 20、30、0、剛好通過門檻的數字。

10. 重點總結與下一步

這份 assessment 的核心，不只是學會寫 if，而是學會把題目規則翻譯成有順序、有邊界意識的程式流程。完成這組題目後，學生應該能更穩定地處理分類題、資格審核題、分段計價題，也能更清楚知道什麼情況要先排除、什麼情況要再分類。

下一步通常會接到迴圈或更長的資料處理流程。到那個階段，條件判斷不會消失，而是會和 for、累加、計數結合，所以這一組的重點其實是後續所有控制流程題目的基礎。
