自動批閱 hw04: For Loop with Range

題組簡介

這一組題目聚焦在 Python 的 for 迴圈與 range()。學生會從最簡單的固定次數輸出，逐步走到區間累加、條件過濾、重複輸入、巢狀迴圈，以及最後的文字圖表輸出。它的教學重點是讓學生理解：當程式需要「重複做某件事」時，不應該手動複製程式碼，而是把相同邏輯交給迴圈。

對初學者來說，這一組真正困難的地方通常不是語法本身，而是搞清楚每一輪迴圈正在處理什麼、累加器或計數器何時更新、range() 到底會產生哪些數字。只要這些基本觀念不穩，後面的統計題和巢狀迴圈題就很容易混亂。

本篇涵蓋重點

1. 題型分類：固定次數輸出、區間過濾、累加統計、重複輸入、巢狀迴圈、文字圖表。

2. 核心概念：for、range(stop)、range(start, stop, step)、累加器、計數器、雙層迴圈。

3. 常見卡點：stop 不包含、初始值設錯、迴圈次數與輸入筆數對不起來、列印格式錯誤。

4. 解題策略：先確認迴圈範圍，再決定每輪要做的事，最後處理輸出格式。

2. 核心概念群一：for 迴圈與 range() 的基本模型

A. 定義式解釋

for 迴圈的意思是：按照一組順序資料，一個一個拿出來處理。range() 則是最常見的整數序列產生器，會提供一串依規則變化的數字給 for 使用。

B. 題目脈絡式解釋

lab01 和 lab02 都在教同一件事：不是你要印幾次就複製幾次 print()，而是讓迴圈自動重複。lab02 再往前一步，讓學生知道 range() 不只會從 0 開始，也可以指定起點和步長。

C. 解題策略式解釋

每次寫 for 時，先回答三個問題：會跑幾輪、迴圈變數每輪代表什麼、每輪要輸出或更新什麼。如果這三件事說不清楚，程式通常還沒準備好開始寫。

D. 常見錯誤與修正

- range(1, n) 以為會包含 n，其實不會。

- 看見 range(n) 就忘了它從 0 開始。

- 把 step 寫錯，導致輸出數列不符合題目。

n = int(input())

for value in range(1, n + 1):
    print(value)

這段在做的是從 1 印到 n。為什麼不是 range(n)？因為題目要的是人類習慣的 1 到 n，不是從 0 開始。初學者最容易看不懂的是 n + 1，原因就在於 range() 的結尾不包含在內。

3. 核心概念群二：迴圈中的條件過濾、累加與計數

A. 定義式解釋

累加器是用來持續加總的變數，常見初始值是 0。計數器是用來記錄符合條件次數的變數，也常從 0 開始。條件過濾則是在每輪迴圈判斷目前資料是否值得處理。

B. 題目脈絡式解釋

lab03、prob01、prob02 都在訓練這組能力。你不是只把數字印出來，而是要在走訪區間時篩掉不需要的數字，並留下總和與個數。

C. 解題策略式解釋

先建立迴圈範圍，再把「符合條件時要做什麼」寫進 if 裡。通常動作只有兩種：更新總和，或更新數量。如果題目兩者都要，就兩個變數一起維護。

D. 常見錯誤與修正

- 忘了把 total、count 初始化成 0。

- if 條件寫對了，但更新變數放在條件外面。

- 邊界區間 [a, b] 少算 b。

a, b = map(int, input().split())
total = 0
count = 0

for number in range(a, b + 1):
    if number % 2 == 1:
        total += number
        count += 1

print(total)
print(count)

這段是在找區間中的奇數總和與奇數個數。每輪先看目前 number 是否為奇數，若是才更新。初學者常錯在把 count += 1 放在 if 外面，結果變成統計全部數字。

4. 核心概念群三：重複輸入與批次資料處理

A. 定義式解釋

當題目先給一個數量 n，再要求讀入 n 筆資料時，這就是典型的批次資料處理。程式要用迴圈重複讀取、重複更新統計值。

B. 題目脈絡式解釋

prob03 和 extra01 都是這種題型。它們的重點不是某一筆成績，而是整批資料的總分、平均、及格人數、最高分或區間分布。

C. 解題策略式解釋

寫這類題目前，先列出每讀到一筆資料要更新哪些東西。例如總分要加、及格數可能加、最高分可能改、區間計數器其中一個可能加。把這些更新點列清楚，主程式就不會亂。

D. 常見錯誤與修正

- 讀到 n 後，忘了真的跑 n 輪。

- 最高分初始化不當，導致全部分數都低於初值時出錯。

- 平均數在迴圈內就提早計算，邏輯變亂。

n = int(input())
total = 0
passed = 0
highest = -1

for _ in range(n):
    score = int(input())
    total += score
    if score >= 60:
        passed += 1
    if score > highest:
        highest = score

這段在做的是每讀一筆成績，就同步更新三種統計。為什麼 highest 可以先設成 -1？因為題目保證分數在 0 到 100，所以第一筆成績一定會比它大。新手常看不懂的是：有些變數每輪都更新，有些只在條件成立時更新。

5. 核心概念群四：巢狀迴圈與圖形輸出

A. 定義式解釋

巢狀迴圈就是在一個迴圈裡再放另一個迴圈。外層通常控制列數或大區塊，內層控制每一列中的欄位或細節。

B. 題目脈絡式解釋

prob04 的星號方框與 prob05 的乘法三角形，都是典型的雙層迴圈題。前者用來處理「列 x 欄」位置判斷，後者用來處理「每列長度不同」的遞增結構。

C. 解題策略式解釋

先只想外層一輪代表什麼，再想內層一輪代表什麼。若這兩個角色分不清楚，雙層迴圈一定會亂。實作時也建議先把一列內容組好，再一次輸出，格式更容易控制。

D. 常見錯誤與修正

- 內外層角色顛倒，輸出形狀錯掉。

- 沒處理每列結尾的換行。

- 乘法三角形每列之間的空白格式不一致。

n = int(input())

for row in range(n):
    line = ""
    for col in range(n):
        if row == 0 or row == n - 1 or col == 0 or col == n - 1:
            line += "*"
        else:
            line += "."
    print(line)

這段是在產生星號方框。row 代表目前第幾列，col 代表目前第幾欄。只要在邊界上就放 *，否則放 .。初學者最常卡住的是不知道該在何時 print()，這題最穩的做法就是先把整列字串拼好，再印出。

6. sample code 與 code snippets 導讀

A. 三參數 range()

for value in range(start, stop, step):
    print(value)

這段是在依照指定起點、終點與步長輸出數字。白話說，就是從 start 開始，每次加 step，但不超過 stop 前就停止。初學者容易誤會 stop 也會被印出，其實不會。

B. 累加器與計數器

total = 0
count = 0

for number in range(1, n + 1):
    if number % 3 == 0:
        total += number
        count += 1

這段在做的是把符合條件的數字加總，同時計算有幾個。為什麼要兩個變數？因為「總和」和「個數」是不同資訊。新手容易只顧著加總，忘了還需要另外計數。

C. 依 n 重複讀資料

n = int(input())

for _ in range(n):
    score = int(input())
    print(score)

這段是在示範如何依照題目給的筆數，重複讀資料。_ 的意思是這輪次數本身不重要，我們只在意它會執行 n 次。初學者常不懂 _，其實它只是慣例上的占位名稱。

D. 用字串重複產生星號

stars = "*" * count
print(f"60-69: {stars}")

這段是在依照人數產生長條圖。"*" 乘上一個整數，會得到重複那麼多次的字串。新手常忽略題目要求，即使沒有星號也要保留冒號後的空白。

8. 解題前必備知識與能力檢核

A. 已會

- 能用 int(input()) 讀入整數。

- 知道 for 會一輪一輪執行縮排區塊。

- 知道 % 可用來判斷奇偶或倍數。

B. 還需補強

- range() 的結尾不包含這件事。

- 累加器、計數器的初始化。

- 巢狀迴圈中「列」與「欄」的分工。

- 平均數、最高分、區間統計的更新時機。

C. 容易誤解

- 以為 range(1, n) 會跑到 n。

- 以為每輪都應該重新把 total 設成 0。

- 以為巢狀迴圈一定很難，其實只是兩層單層迴圈的組合。

9. 解題引導與學習建議

這組題目的解法幾乎都可以用同一個模板思考：先決定迴圈跑幾輪，再決定每輪要檢查什麼條件，最後決定每輪要更新哪些變數或輸出什麼內容。只要把「範圍、條件、動作」三件事說清楚，程式通常就不會偏掉。

實作時建議：

1. 先寫最小可執行版本，例如先把區間全部印出來。

2. 再加入 if 做過濾。

3. 最後才加入累加、計數、格式控制。

除錯時建議：

- 印出目前迴圈變數，確認範圍正確。

- 印出 total 或 count，確認是否只在正確時機更新。

- 對雙層迴圈先測最小案例，例如 n = 2 或 n = 3。
